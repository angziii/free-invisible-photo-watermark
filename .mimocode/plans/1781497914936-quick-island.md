# 鲁棒性增强实施方案

## 当前问题
DWT+DCT+SVD+QIM 水印方案对 JPEG 压缩、旋转、格式转换脆弱，仅靠重复+平均无纠错能力。

---

## 改进方案（按影响力排序）

### 1. BCH 纠错编码（最高优先级）
**影响**：将"脆弱"变为"部分抵抗"
**改动**：
- 新建 `blind_watermark/bch_codec.py`，实现 BCH(31,16) 编码器（纯 Python，无新依赖）
- `blind_watermark/blind_watermark.py` 的 `read_wm()` 中对水印 bit 进行 BCH 编码
- `extract()` 中解码纠错
- 添加 `use_ecc=True` 参数，默认开启，向后兼容

### 2. 加权平均提取（简单高效）
**影响**：提取准确率提升 5-15%
**改动**：
- 修改 `bwm_core.py` 的 `extract_avg()` 方法
- 按每个 bit 距离 0.5 的远近赋权，高置信度块权重更大

### 3. 自适应 d1/d2 参数
**影响**：同时改善不可见性和鲁棒性
**改动**：
- `bwm_core.py` 新增 `compute_block_d()` 方法，基于块能量（DCT 系数 RMS）缩放 d1/d2
- 高能量块（纹理/边缘）→ 更大 d → 更鲁棒（视觉上被内容掩盖）
- 低能量块（平滑区域）→ 更小 d → 更少伪影
- 添加 `adaptive=True` 参数

### 4. JPEG 感知嵌入
**影响**：显著提升 JPEG 压缩下的水印存活率
**改动**：
- `bwm_core.py` 新增 `apply_jpeg_aware_mask()` 函数
- 在 SVD 前对 DCT 系数进行频率掩码，保留 JPEG 会保留的中低频分量
- 仅嵌入 JPEG 量化表不会清零的频段
- 添加 `jpeg_aware=True` 参数

### 5. 抗旋转（小角度）
**影响**：支持 ±15° 旋转恢复
**改动**：
- `blind_watermark/recover.py` 新增 `estimate_rotation_angle()` 函数
- 暴力搜索角度范围，按提取置信度选最优角度
- `blind_watermark.py` 添加 `auto_rotate=False` 参数

### 6. 浏览器 JS 同步更新
**改动**：`assets/watermark-worker.js` 同步实现上述所有改进

## 实施顺序与工期
| 阶段 | 内容 | 工期 | 依赖 |
|------|------|------|------|
| 1 | BCH 纠错编码 | 2-3 天 | 无 |
| 2 | 加权平均 | 0.5 天 | 无 |
| 3 | 自适应 d1/d2 | 1 天 | 无 |
| 4 | JPEG 感知嵌入 | 1 天 | 无 |
| 5 | 抗旋转估计 | 2 天 | 阶段 1 |
| 6 | JS 同步 | 2-3 天 | 阶段 1-4 |

**最小可行改进**（80% 增益）：阶段 1 + 2 + 3 = 3.5-4.5 天

## 验证方案
```bash
# 运行现有测试确保向后兼容
python -m pytest tests/

# 运行新鲁棒性测试
python -m pytest tests/test_robustness.py -v

# 手动验证：嵌入→JPEG压缩→提取
python -c "
from blind_watermark import WaterMark, att
bwm = WaterMark(password_img=1, password_wm=1, use_ecc=True, adaptive=True)
bwm.read_img('test.jpg')
bwm.read_wm('Hello', mode='str')
bwm.embed()
bwm.output('watermarked.png')
# JPEG 压缩后再提取
att.jpeg_attack('watermarked.png', './tmp/', compression_ratio=50)
bwm2 = WaterMark(password_img=1, password_wm=1, use_ecc=True, adaptive=True)
wm = bwm2.extract('watermarked_jpeg.png', wm_shape=(1, len('Hello')*8))
print(wm)
"
```
